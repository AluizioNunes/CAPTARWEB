PROBLEMA 1: "Failed to resolve import ./pages/Dashboard"
═══════════════════════════════════════════════════════════════════════════════

Causa:
  - O arquivo era DashboardPage.tsx, nao Dashboard.tsx
  - App.tsx estava importando o arquivo errado
  - Vite nao conseguia encontrar o arquivo

Solucao:
  - Corrigir import em App.tsx
  - De: import Dashboard from './pages/Dashboard'
  - Para: import DashboardPage from './pages/DashboardPage'
  - Usar: <DashboardPage /> na rota

Por que demorou a resolver:
  - Vite usa cache em memoria
  - Mesmo apos corrigir o arquivo, o erro persistia
  - Necessario parar o servidor e limpar cache

═══════════════════════════════════════════════════════════════════════════════

PROBLEMA 2: Porta 3000 vs 5173
═══════════════════════════════════════════════════════════════════════════════

Informacao:
  - Vite por padrao usa porta 5173
  - Neste projeto, foi configurado para usar porta 3000
  - Isso foi feito propositalmente em vite.config.ts

Motivo:
  - Facilitar acesso local (porta comum)
  - Compatibilidade com configuracoes existentes
  - Proxy para API em localhost:8000

Configuracao em vite.config.ts:
  server: {
    port: 3000,
    host: '0.0.0.0',
    proxy: {
      '/api': {
        target: 'http://localhost:8000',
        changeOrigin: true,
        rewrite: (path) => path.replace(/^\/api/, '')
      }
    }
  }

Se quiser mudar para 5173:
  - Editar vite.config.ts
  - Mudar: port: 3000 para port: 5173
  - Reiniciar servidor

PROBLEMA 3: Cache do Vite
═══════════════════════════════════════════════════════════════════════════════

O que e:
  - Vite armazena cache em memoria durante desenvolvimento
  - Permite Hot Module Replacement (HMR) mais rapido
  - Mas pode causar problemas se arquivo foi deletado/movido

Como limpar:
  1. Parar servidor: Ctrl+C
  2. Deletar cache: Remove-Item -Recurse -Force node_modules\.vite
  3. Reiniciar: npm run dev

Ou mais simples:
  1. Parar servidor: Ctrl+C
  2. Reiniciar: npm run dev

DIFERENCA: Streamlit v1.0 vs Vite v2.0
═══════════════════════════════════════════════════════════════════════════════

Streamlit v1.0:
  - Python backend
  - Streamlit frontend (Python)
  - Sem TypeScript
  - Sem React
  - Sem Vite

Vite v2.0:
  - FastAPI backend (Python)
  - React frontend (TypeScript)
  - Vite como bundler
  - Hot Module Replacement (HMR)
  - Compilacao TypeScript em tempo real
  - Melhor performance
  - Melhor experiencia de desenvolvimento

COMO FUNCIONA O VITE
═══════════════════════════════════════════════════════════════════════════════

1. Desenvolvimento (npm run dev):
   - Vite inicia servidor em porta 3000
   - Carrega modulos sob demanda

   - Minifica codigo
   - Gera arquivos otimizados em dist/
   - Pronto para producao

3. Preview (npm run preview):

   - Testa build antes de fazer deploy
